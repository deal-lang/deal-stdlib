§1 Dependency decision — N-07 is unblocked

Split N-07; move Torque≠Energy enforcement out of Lane B.

  • Check #7 compares only the 7-exponent vector, so the compiler already treats Torque and Energy (both M·L²·T⁻²) as identical. Declaring them as distinct named dimensions is pure content that parses + checks today.
  • Enforcing non-interchange needs a named-type tie-break in sema.zig — a Lane A feature gated on N-00③, reclassified out of Lane B.
  • Common products (Voltage*Current=Power) already resolve via vector addition in checkBinaryDimension — no dimension-equation table needed.

Net: Lane B has zero blocking dependencies and starts today. N-07 declares same-vector pairs distinctly and tags each with @same_vector_note so the later Lane A pass has an exact enforcement list.

§4 Encoding (locked — Option A)

// dimension def: all seven si_M..si_J exponents, no <<specializes>>
attribute def Pressure {
    attribute si_M  : Integer = 1;   attribute si_L  : Integer = -1;
    attribute si_T  : Integer = -2;  attribute si_I  : Integer = 0;
    attribute si_TH : Integer = 0;   attribute si_N  : Integer = 0;
    attribute si_J  : Integer = 0;
}
// unit def: <<specializes>> a dimension + si_factor
attribute def Pa  <<specializes>> Pressure { attribute si_factor : Real = 1.0; }
attribute def kPa <<specializes>> Pressure { attribute si_factor : Real = 1000.0; }
// same-vector name kept distinct, flagged for Lane A enforcement
// @same_vector_note: distinct-from Energy — Lane A enforcement (N-00 ③)
attribute def Torque { /* M=1 L=2 T=-2, rest 0 */ }
